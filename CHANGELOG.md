## 0.1.17 - 2026-05-22

- В акт инвентаризации добавлен выбор ЦФО.
- В акт инвентаризации добавлен выбор места / зоны на базе складов: кухня, бар, зал, склад и другие зоны.
- Добавление оборудования в акт теперь учитывает выбранные локацию, ЦФО и место / зону.

## 0.1.16 - 2026-05-21

- Добавлены русские статусы инвентаризации: Активная, Проведение - согласование, Подтверждено, Отменена.
- В актах инвентаризации ответственный и подтверждающий показываются по ФИО, если оно заполнено в профиле пользователя.
- В списке инвентаризаций добавлены выпадающие фильтры по юрлицу и локации.
- Старые статусы актов автоматически переводятся на новую схему при миграции.

## 0.1.15 - 2026-05-20

- Добавлен fallback для сканирования QR на Safari/iPhone через локальный jsQR.
- Камера больше не зависит только от браузерного `BarcodeDetector`.
- В сообщениях подсказано, что на iPhone доступ к камере требует HTTPS или разрешения камеры в браузере.

## 0.1.14 - 2026-05-20

- Сканирование в акте инвентаризации теперь поддерживает NFC-метки.
- Оборудование можно найти по NFC UID, коду метки, ссылке из метки или UUID оборудования.
- Интерфейс сканирования переименован в QR/NFC, чтобы было понятно, что работают оба способа.

## 0.1.13 - 2026-05-20

- В акт инвентаризации добавлено поле для сканирования QR-кода оборудования.
- Сканирование принимает полную QR-ссылку, локальный путь, код метки или UUID оборудования.
- На поддерживаемых браузерах можно включить камеру и считать QR прямо со страницы акта.
- После сканирования оборудование сразу добавляется в текущий акт, а повторный скан обновляет отметку без дублей.

## 0.1.12 - 2026-05-20

- В акт инвентаризации теперь можно добавить активное оборудование выбранной локации, даже если юрлицо оборудования отличается от юрлица акта.
- В списке добавления позиции отображаются название, инвентарный номер, юрлицо и склад.
- Уже добавленные в акт позиции больше не показываются повторно в выпадающем списке.

## 0.1.11 - 2026-05-20

- Форма создания и редактирования юридических лиц во внутренней админ-панели переведена на русский язык.
- Поля `name`, `short_name`, `tax_id`, `is_active` теперь отображаются как "Название", "Краткое название", "ИНН", "Активно".

## 0.1.10 - 2026-05-20

- Добавлено версионирование CSS и JS в URL, чтобы iPad не держал старую статику формы.
- Исправление поля "Склад" теперь принудительно доходит до браузера после обновления версии.

## 0.1.9 - 2026-05-20

- Исправлена адаптивность формы оборудования на iPad: длинные подписи дат больше не накладываются друг на друга.
- Раздел файлов оборудования переименован в "Документы".
- В карточку оборудования добавлена быстрая кнопка "Добавить документ".

## 0.1.8 - 2026-05-20

- Исправлена битая кодировка подсказки в поле "Склад" на форме оборудования.
- Выпадающий список склада теперь показывает "Выберите склад" нормальными русскими буквами.

## 0.1.7 - 2026-05-20

- Исправлено открытие только что созданного пустого акта инвентаризации для администратора.
- Проверены серверные backup-логи: локальный backup работает, Google Drive и GitHub пока не подключены в конфигурации.

## 0.1.6 - 2026-05-15

- Автоматический backup переведен на расписание раз в 10 дней.
- В документации явно описан ручной запуск backup.
- Страница версий обновлена под новый режим резервирования.

## 0.1.5 - 2026-05-15

- Добавлена автоматическая production-логика backup в три резерва: сервер, Google Drive и GitHub.
- Cloud/GitHub backup разрешен только для зашифрованного архива.
- Добавлены пример конфигурации backup и установщик cron.

## 0.1.4 - 2026-05-15

- Backup базы переведен в стандартный XML-формат фикстур Django.
- Добавлена команда `backup_database_xml` для повторяемого полного XML backup.
- Скрипты backup/restore теперь сохраняют `database.xml` и восстанавливают базу через `loaddata`.

## 0.1.3 - 2026-05-15

- Исправлена ошибка 500 при создании нового пользователя через внутреннюю админ-панель.
- Новый пользователь теперь корректно получает пароль и выбранные права доступа.

## 0.1.2 - 2026-05-15

- Русифицированы формы и основные пользовательские подписи интерфейса.
- Роли, уровни доступа и подписи секретов теперь отображаются на русском языке.
- Английские названия в меню и таблицах заменены на понятные русские формулировки.

## 0.1.1 - 2026-05-15

- Добавлена страница "Версии патчей" с понятным описанием изменений.
- Во вкладке пользователя "Права доступа" юрлица и локации выбираются списком галочек.
- Исправлено сохранение пользователя: пустое поле нового пароля больше не сбрасывает старый пароль.

## 0.1.0 - 2026-05-14

- Подготовлена первая серверная линия системы оборудования.
- Добавлен production Docker compose с Gunicorn и Nginx.
- Добавлены настройки безопасности для `DJANGO_DEBUG=False`.
- В интерфейсе отображается версия приложения.
- Описан процесс выпуска патчей и серверного деплоя.
